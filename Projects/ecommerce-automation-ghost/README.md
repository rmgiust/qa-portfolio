# [E2E] Automation: Critical Purchase Flow | Automação de Fluxo de Compra

## 🎯 Objective | Objetivo
**EN:** This project demonstrates an automated End-to-End (E2E) testing suite for the **Automation Exercise** platform. [cite_start]The goal was to validate the critical business path: User Login, Product Selection, and Checkout completion[cite: 4, 12].

**PT:** Este projeto demonstra uma suíte de testes automatizados End-to-End (E2E) para a plataforma **Automation Exercise**. [cite_start]O objetivo foi validar o caminho crítico do negócio: Login, Seleção de Produto e Finalização de Checkout[cite: 4, 12].

---

## 🛠️ Tools & Technologies | Ferramentas e Tecnologias
* [cite_start]**Ghost Inspector:** Cloud-based No-Code Automation & Visual Testing[cite: 5].
* [cite_start]**XPath:** Advanced element targeting for resilient selectors[cite: 42].
* [cite_start]**Visual Regression:** UI stability validation with a 10% tolerance threshold[cite: 43].

---

## 🚀 Technical Highlights | Destaques Técnicos

### 1. Resilient Selector Strategy | Estratégia de Seletores Resilientes
[cite_start]**EN:** Implemented robust **XPath** selectors to handle dynamic content and nested HTML tags, significantly reducing test "brittleness"[cite: 42].
[cite_start]**PT:** Implementação de seletores **XPath** robustos para lidar com conteúdo dinâmico e tags HTML aninhadas, reduzindo significativamente a fragilidade dos testes[cite: 42].

### 2. Visual Regression Testing | Testes de Regressão Visual
[cite_start]**EN:** Integrated automated UI comparisons to ensure layout stability across different browser sessions[cite: 43].
[cite_start]**PT:** Integração de comparações automatizadas de interface (UI) para garantir a estabilidade do layout entre diferentes sessões do navegador[cite: 43].

### 3. Automated Validation (Assertions) | Validação Automatizada (Asserções)
[cite_start]**EN:** Combined functional assertions (verifying "Logged in as" status) with end-of-flow confirmation messages to eliminate false positives[cite: 45].
[cite_start]**PT:** Combinação de asserções funcionais (verificando o status "Logged in as") com mensagens de confirmação de fim de fluxo para eliminar falsos positivos[cite: 45].
